## 各 Agent 详细说明

### 1. 日报 Agent —— 每日自动闭环

**做什么**：每天固定时间自动生成运营日报，推送到飞书

**数据来源**：RPA 采集的 Excel（90+ 字段）+ 浏览器自动采集

**输出示例**：

```
┌─────────────────────────────────────────────────────┐
│  📊 每日运营日报 · 2026-06-09                        │
│  ───────────────────────────────────────────────── │
│  运营人员：张三                                      │
│  负责门店：趣东北·东北小馆(石岩店)                     │
│  ───────────────────────────────────────────────── │
│                                                     │
│  ┌──────────┬──────────┬──────────┬──────────┐     │
│  │ 核心指标   │ 昨日      │ 前日      │ 环比      │     │
│  ├──────────┼──────────┼──────────┼──────────┤     │
│  │ 营业额    │ ¥2,350   │ ¥2,180   │ +7.8% ↑  │     │
│  │ 有效订单  │ 57       │ 52       │ +9.6% ↑  │     │
│  │ 单均实付  │ ¥25.82   │ ¥24.50   │ +5.4% ↑  │     │
│  │ 曝光人数  │ 1,985    │ 1,820    │ +9.1% ↑  │     │
│  │ 进店转化  │ 10.6%    │ 10.1%    │ +0.5pp ↑ │     │
│  │ 下单转化  │ 26.5%    │ 25.8%    │ +0.7pp ↑ │     │
│  └──────────┴──────────┴──────────┴──────────┘     │
│                                                     │
│  ⚠️ 预警：差评 0 条 · 出餐超时 16 单 · 商责退单 0     │
│  📌 今日待办：菜品图更新(3个) · 活动到期续签(1个)      │
│  ───────────────────────────────────────────────── │
│  [📈 详细分析]  [📋 生成周报]  [🏪 门店巡检]         │
└─────────────────────────────────────────────────────┘
```

**核心字段**（基于 RPA 数据）：
- 营收类：营业额、有效订单、单均实付、单均收入
- 流量类：曝光人数、进店人数、下单人数
- 转化类：进店转化率、下单转化率、新客/老客转化率
- 质量类：差评数、投诉数、出餐超时、商责退单率
- 活动类：活动订单占比、投入产出比、营销力度
- 复购类：7日复购率、30日复购率

### 2. 预警 Agent —— 实时监控，主动推送

**做什么**：基于预设规则，自动扫描数据异常，飞书推送预警

**预警规则**（可配置）：

| 指标 | 预警条件 | 级别 |
|------|---------|------|
| 营业额 | 环比下降 >15% | 🔴 严重 |
| 营业额 | 环比下降 >10% | 🟡 警告 |
| 评分 | 下降至 4.5 以下 | 🔴 严重 |
| 差评 | 单日新增 >3 条 | 🟡 警告 |
| 曝光人数 | 连续 3 天下降 | 🟡 警告 |
| 转化率 | 环比下降 >3pp | 🟡 警告 |
| 商责退单率 | 超过 5% | 🔴 严重 |
| 出餐超时 | 超过 20 单 | 🟡 警告 |

**预警卡片**：

```
┌─────────────────────────────────────────────────────┐
│  🔴 营业额预警 · 趣东北(石岩店)                       │
│  ───────────────────────────────────────────────── │
│  预警内容：营业额环比下降 18.5%                        │
│  昨日：¥2,350 → 前日：¥2,860（差额 ¥510）            │
│                                                     │
│  🤖 AI 初步分析：                                    │
│  • 曝光人数下降 12%，是主要下滑因素                    │
│  • 进店转化率基本持平，非页面问题                      │
│  • 昨日为周日，前日为周六，存在周期性波动               │
│  • 建议：观察今日数据，若持续下滑需检查推广投放         │
│  ───────────────────────────────────────────────── │
│  [📈 深度诊断]  [📊 查看趋势]  [✅ 已知晓]           │
└─────────────────────────────────────────────────────┘
```

### 5. 增长 Agent —— 数据诊断，优化建议

**做什么**：当门店数据下滑时，自动排查原因，输出可落地的优化建议

**诊断流程**：

```
门店数据下滑触发
  ↓
Step 1: 定位核心下滑指标
  → 营业额下滑 → 拆解为：曝光 × 进店转化 × 下单转化 × 客单价
  ↓
Step 2: 逐层排查
  → 曝光下滑？→ 检查排名、活动、推广
  → 转化下滑？→ 检查评分、差评、菜单、价格
  → 客单价下滑？→ 检查菜品结构、满减设置
  ↓
Step 3: 结合知识库 SOP 输出建议
  → "曝光下降，建议检查：1.是否活动到期 2.是否被竞品抢排名..."
  ↓
Step 4: 生成行动清单
  → 飞书卡片推送待办事项
```

### 6. 知识库 Agent —— 运营 SOP 智能问答

**做什么**：团队内部的运营知识问答机器人

**对话示例**：

```
运营："新店开业前 7 天应该做什么？"

AI："📋 新店开业 7 天运营 SOP：

Day 1-3（冷启动期）：
• 设置新客立减活动（满 20 减 5）
• 菜品定价略低于竞品 5-10%
• 确保 3 张以上精修菜品图
• 营业时间设置为全时段

Day 4-7（爬坡期）：
• 逐步提升曝光投放预算
• 引导前 3 天好评（随餐附好评卡）
• 优化差评回复（24 小时内回复）
• 监控转化率，低于 15% 需优化菜单

📌 详细 SOP 文档：[点击查看飞书文档]"
```

**知识库内容**：
- 运营 SOP（新店、日常、活动、差评处理）
- 平台规则（美团、饿了么）
- 话术库（差评回复、客户沟通）
- 定价策略、菜单优化方法论

---

### 7. 菜单 Agent —— 菜品数据分析与优化

**做什么**：基于菜品数据，分析动销、优化菜单结构、定价测算

**核心能力**：
- 菜品动销分析（哪些卖得好、哪些滞销）
- 菜单结构优化（品类占比、价格带分布）
- 定价测算（基于竞品和成本）
- 爆品打造建议
- 滞销菜品处理方案

## 与竞品的差异

| 维度 | 通用 AI 助手 | RPA 工具 | 点之出众 |
|------|------------|---------|---------|
| 定位 | 通用对话 | 流程自动化 | 餐饮代运营 AI |
| 能力 | 能聊天，不懂业务 | 能搬数据，不能分析 | 采集+分析+预警+执行+学习 |

| 执行 | 不能 | 能但无智能 | 智能执行+人工确认 |
| 学习 | 不沉淀 | 不沉淀 | 知识库+记忆自动沉淀 |
| 入口 | 独立 APP | 独立软件 | 飞书（团队已有工具） |

## 总结

**点之出众 = 飞书入口 + AI Agent + 餐饮运营知识**

- 用**飞书**做入口——运营团队已经在用，零迁移成本
- 用 **AI Agent** 做执行——替代 80% 重复性工作
- 用**餐饮运营知识**做决策——不是通用 AI，是懂行的 AI

目标：让 10 人团队管好 100 家门店，人效提升 5-10 倍。
