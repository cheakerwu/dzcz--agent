# 飞书能力与餐饮代运营入口方案

> 目标：以飞书作为餐饮代运营的统一入口，充分利用飞书开放平台能力，让 AI 助手不只是"聊天机器人"，而是"可交互的运营面板"。

---

## 三、最有价值的飞书能力（按优先级）

### 🥇 P0：消息卡片（立即可用，体验提升最大）

消息卡片是飞书最强大的交互能力。AI 不只是回复文字，而是呈现一个**可交互的运营面板**。

#### 场景示例：每日营业数据卡片

```
┌─────────────────────────────────────────┐
│  📊 今日营业数据 - 门店A                  │
│  ─────────────────────────────────────── │
│  营业额: ¥3,280  │  订单数: 86          │
│  客单价: ¥38.1   │  评分: 4.7 ⭐        │
│  ─────────────────────────────────────── │
│  ⚠️ 差评预警: 2 条新差评                  │
│  ─────────────────────────────────────── │
│  [📈 详细分析]  [📝 回复差评]  [📋 生成报告] │
│  [🏪 门店巡检]  [⏰ 设置定时]             │
└─────────────────────────────────────────┘
```

#### 技术实现

**卡片 JSON 结构**：
```json
{
  "config": { "wide_screen_mode": true },
  "header": {
    "title": { "tag": "plain_text", "content": "📊 今日营业数据" },
    "template": "blue"
  },
  "elements": [
    {
      "tag": "div",
      "fields": [
        { "is_short": true, "text": { "tag": "lark_md", "content": "**营业额**\n¥3,280" }},
        { "is_short": true, "text": { "tag": "lark_md", "content": "**订单数**\n86" }}
      ]
    },
    { "tag": "hr" },
    {
      "tag": "action",
      "actions": [
        {
          "tag": "button",
          "text": { "tag": "plain_text", "content": "📈 详细分析" },
          "type": "primary",
          "value": { "action": "detail_analysis", "store": "store-a", "date": "today" }
        },
        {
          "tag": "button",
          "text": { "tag": "plain_text", "content": "📝 回复差评" },
          "type": "default",
          "value": { "action": "reply_reviews", "store": "store-a" }
        }
      ]
    }
  ]
}
```

**卡片回调流程**：
```
用户点击按钮
  → 飞书服务器 POST 到回调 URL
  → DeepBot 接收 action.value
  → 路由到对应处理逻辑
  → 执行任务（Agent 推理 或 直接处理）
  → 返回新卡片 JSON（更新卡片内容）
```

**回调数据示例**：
```json
{
  "open_id": "ou_xxx",
  "open_message_id": "om_xxx",
  "action": {
    "value": { "action": "collect_data", "store": "store-a" },
    "tag": "button"
  }
}
```

### 🥈 P1：多维表格（数据沉淀，运营核心）

多维表格是飞书版的"Airtable"，天然适合餐饮运营数据管理。

#### 应用场景

| 表格 | 用途 | 数据来源 |
|------|------|---------|
| 营业数据表 | 每日/每周经营数据沉淀 | Agent 自动采集写入 |
| 差评记录表 | 差评归档、趋势分析 | Agent 监控写入 |
| 门店信息表 | 门店基础信息维护 | 人工维护，Agent 读取 |
| 竞品监控表 | 竞品价格/活动跟踪 | Agent 定时采集 |
| 任务跟踪表 | 运营任务进度管理 | Agent + 人工协同 |

#### 营业数据表示例

| 日期 | 门店 | 平台 | 营业额 | 订单数 | 客单价 | 评分 | 差评数 |
|------|------|------|--------|--------|--------|------|--------|
| 06-09 | 门店A | 美团 | ¥3,280 | 86 | ¥38.1 | 4.7 | 2 |
| 06-09 | 门店A | 饿了么 | ¥2,150 | 58 | ¥37.1 | 4.5 | 1 |
| 06-08 | 门店A | 美团 | ¥3,100 | 82 | ¥37.8 | 4.8 | 0 |

#### 需要新增的工具

```typescript
feishu_bitable_create_app      // 创建多维表格
feishu_bitable_list_tables     // 列出数据表
feishu_bitable_create_record   // 写入记录
feishu_bitable_list_records    // 查询记录（支持筛选）
feishu_bitable_update_record   // 更新记录
feishu_bitable_delete_record   // 删除记录
feishu_bitable_create_field    // 创建字段
```

#### 技术参考

飞书多维表格 API：https://open.feishu.cn/document/server-docs/docs/bitable-v1/bitable-overview

### 🥉 P2：知识库（运营知识沉淀）

飞书知识库（Wiki）可以作为团队的运营知识中心，Agent 可以搜索和引用。

#### 应用场景

| 知识库 | 内容 | Agent 用途 |
|--------|------|-----------|
| 运营 SOP 库 | 标准操作流程、培训资料 | 用户问"怎么做"时自动引用 |
| 差评话术库 | 各类差评的标准回复模板 | 回复差评时智能推荐 |
| 菜品信息库 | 菜品介绍、价格、原料 | 查询菜品信息时引用 |
| 供应商信息库 | 供应商联系方式、报价 | 查询供应商信息时引用 |
| 平台规则库 | 美团/饿了么运营规则 | 运营建议时参考 |

#### 需要新增的工具

## 四、飞书卡片回调技术实现

### 回调配置

1. 在飞书开放平台创建应用
2. 进入「事件订阅」→ 配置卡片回调 URL
3. 回调 URL 需要能响应飞书的 challenge 验证

### 回调处理流程

```
飞书服务器
  ↓ POST /api/feishu/card/callback
Express/Connector 接收
  ↓ 解析 action.value
路由分发
  ├─ action = "collect_data"  → Agent 采集数据
  ├─ action = "reply_reviews" → Agent 生成回复
  ├─ action = "detail_analysis" → Agent 分析数据
  ├─ action = "confirm_exec"  → 执行写操作
  └─ action = "cancel"        → 取消操作
  ↓ 返回新卡片 JSON 或 toast 提示
飞书更新卡片
```

### 回调数据结构

```json
{
  "open_id": "ou_xxx",
  "open_message_id": "om_xxx",
  "open_chat_id": "oc_xxx",
  "tenant_key": "xxx",
  "token": "验证token",
  "action": {
    "value": {
      "action": "collect_data",
      "store": "store-a",
      "date": "today"
    },
    "tag": "button"
  }
}
```

### 返回值（更新卡片）

```json
{
  "toast": {
    "type": "success",
    "content": "操作成功"
  }
}
```

或返回完整卡片 JSON 来更新卡片内容。

### 注意事项

- 回调超时时间为 **3 秒**，需要快速返回
- 长时间任务需要先返回 toast，然后通过新消息发送结果
- 卡片模板支持版本管理（Card Template）
